## ✅ 验证要点

### 关键检查项

1. **字段存在性**
   - ✅ `can_use_volume` 字段存在
   - ✅ [raw['available']](file://d:\main_data\quant_env\Lib\site-packages\pytz\zoneinfo\Pacific\Tarawa) 字段存在
   - 两者数值应该一致

2. **T+1逻辑正确性**
   - ✅ 今日买入的股票：`can_use_volume = 0`
   - ✅ 昨日持仓：`can_use_volume = volume`
   - ✅ 混合持仓：`can_use_volume < volume`

3. **与实际持仓对比**
   - 对比掘金界面"可平"列的数值
   - 应该完全一致

## 🔍 常见问题

### 问题1：无法获取持仓数据

**错误提示：**
```
❌ 测试失败: query_positions() 调用失败
```

**解决方案：**
1. 确认掘金终端已登录
2. 确认账户有持仓
3. 检查网络连接
4. 确认token有效

### 问题2：字段不存在

**输出示例：**
```
❌ 可平数量 (can_use_volume): 字段不存在
```

**解决方案：**
1. 检查掘金SDK版本（建议 >= 3.0.183）
2. 查看 `raw` 属性的完整内容
3. 确认实际字段名（可能是 `available`、`can_sell` 等）
4. 更新 [core/trader_engine.py](file://d:\main_data\core\trader_engine.py#L50-L50) 中的字段映射

### 问题3：可平数量与界面不符

**可能原因：**
1. 数据未刷新（等待几秒后重新运行）
2. 字段映射错误
3. 掘金SDK版本问题

**解决方案：**
1. 查看 `raw` 属性中的所有字段
2. 手动对比掘金界面数据
3. 联系掘金技术支持
