## 💡 常见原因

### 1️⃣ **网络波动**（最常见）
- 本地网络与掘金服务器之间的临时中断
- 会自动恢复

### 2️⃣ **非交易时段维护**
- 掘金服务器在开盘前/收盘后进行例行维护
- 当前时间：**2026-04-20 09:58**（接近开盘时间）
- 开盘后通常会恢复正常

### 3️⃣ **Token 权限限制**
- 某些 Token 只有**查询权限**，没有**交易推送权限**
- 但这**不影响**你手动下单或程序化下单

### 4️⃣ **服务器端负载**
- 开盘前大量用户连接，服务器压力大
- 临时拒绝部分推送连接

### 方案 3: **添加自定义错误处理**（已实施）✅

已在 [`main.py`](d:\mpython\main.py) 中添加 [`on_error()`](file://d:\mpython\main.py#L58-L85) 函数：

```python
def on_error(context, error_code, info):
    """自定义交易服务错误处理"""
    log = get_logger()
    if not log:
        return
    
    # 交易消息服务相关错误（1100/1101）通常会自动恢复，降低日志级别
    if error_code in [1100, 1101]:
        # 只在首次出现时记录，避免刷屏
        if not hasattr(on_error, '_last_warning_time'):
            on_error._last_warning_time = 0
        
        current_time = time.time()
        # 每60秒最多记录一次
        if current_time - on_error._last_warning_time > 60:
            on_error._last_warning_time = current_time
            log.log(f'[交易服务] 连接状态变化: code={error_code}, {info} (将自动重连)')
    else:
        # 其他错误正常记录
        log.log(f'[交易服务错误] code={error_code}, info={info}')
```

**效果**：
- ✅ 避免日志刷屏（每60秒最多记录一次）
- ✅ 明确告知"将自动重连"
- ✅ 其他错误仍会正常记录
