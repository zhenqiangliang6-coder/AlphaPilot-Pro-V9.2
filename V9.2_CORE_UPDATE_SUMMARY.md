## ✅ V9.2 主要更新内容

### 1. **修复掘金SDK持仓字段映射错误**

**问题**：`can_use_volume` 使用了错误的 `available` 字段（包含今日买入）

**修复**：
```python
# ❌ 修复前
self.can_use_volume = raw.get("available", 0)

# ✅ 修复后
self.available_now = raw.get("available_now", 0)  # 当前可卖数量
self.can_use_volume = self.available_now  # 兼容字段
```

**影响**：确保T+1合规性100%，今日买入的股票不会被错误卖出。

---

### 2. **修复即时策略卖出未执行100股取整**

**问题**：直接卖出 `pos.can_use_volume`，可能导致非100整数倍订单

**修复**：
```python
# ❌ 修复前
result = self.engine.order_stock(code, "SELL", pos.can_use_volume, ...)

# ✅ 修复后
can_sell = pos.can_use_volume
actual_sell_volume = (can_sell // 100) * 100

if actual_sell_volume == 0 and can_sell >= 100:
    actual_sell_volume = 100

result = self.engine.order_stock(code, "SELL", actual_sell_volume, ...)
```

**影响**：所有卖出操作现在是100股的整数倍，符合A股交易规则。

## ⚠️ 重要提醒

### 严禁行为

- ❌ **禁止使用 `available` 字段进行卖出决策**（会违反T+1规则）
- ❌ **禁止直接使用 `volume` 计算卖出数量**
- ❌ **禁止绕过 `can_use_volume` 检查**
- ❌ **禁止卖出非100整数倍的数量**（清仓除外）
- ❌ **禁止跳过接口测试直接运行策略**

### 推荐做法

- ✅ **所有卖出操作必须基于 `available_now` 或 `can_use_volume`**
- ✅ **卖出前检查 `can_sell <= 0` 并跳过**
- ✅ **卖出数量向下取整为100的整数倍**
- ✅ **日志显示"总持仓"和"可卖数量"的对比**
- ✅ **切换平台前必须先测试接口字段**
