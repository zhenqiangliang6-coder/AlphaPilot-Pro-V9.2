## 📋 一、需求背景

### 1.1 业务需求
科创板（68开头）和创业板（30开头）股票具有更高的波动性，原有的统一止盈止损策略无法充分发挥其盈利潜力，同时可能因止损过严导致频繁触发。

### 1.2 核心目标
- **止损优化**：为30/68开头股票提供更宽松的止损空间（-1.6%/-3.5% vs -1.2%/-2.5%）
- **止盈优化**：为30/68开头股票提供更大的盈利空间（回落1.5%/上限17% vs 回落1.3%/上限8.5%）
- **架构一致性**：严格遵循"决策与执行分离"原则，通过配置化实现差异化策略

## 🏗️ 二、系统级架构设计

### 2.1 设计原则
1. **配置驱动**：所有阈值参数从配置文件读取，支持灵活调整
2. **动态选择**：根据股票代码前缀自动选择对应策略参数
3. **前缀兼容**：支持掘金格式（SZSE.300444）、外部信号格式（300444.SZ）、纯数字格式
4. **T+1合规**：完整保留可卖数量检查逻辑，确保A股交易合规

### 2.2 技术实现路径
```
信号输入 → 代码前缀识别 → 参数动态选择 → 风控执行 → T+1合规检查 → 下单执行
```

## ⚠️ 七、风险提示

### 7.1 技术风险
- **前缀识别错误**：已通过 `_extract_numeric_code()` 方法全面兼容多种格式
- **参数未加载**：使用 `getattr()` 提供默认值，避免KeyError
- **T+1违规**：完整保留 `can_use_volume` 检查，确保合规

### 7.2 市场风险
- **过度宽松止损**：-3.5%清仓可能导致单笔亏损扩大，建议配合仓位管理
- **止盈上限提高**：17%上限可能延长持仓时间，增加不确定性
- **波动性误判**：非30/68股票若出现高波动，仍使用原有严格策略

### 7.3 应对策略
1. **止损放大后的风险控制**：
   - 建议降低30/68股票的初始仓位（如从10%降至7%）
   - 启用动态仓位管理模块（Rocket Boost）的保守模式

2. **止盈优化的收益保护**：
   - 密切观察第二/三级止盈的触发频率
   - 如发现频繁触及17%上限后大幅回落，可考虑调整为15%

3. **持续监控指标**：
   - 30/68股票的平均持仓时长
   - 止损触发率 vs 普通股票的对比
   - 止盈完成率（成功落袋的比例）
