## 🎯 核心发现与修复

### 问题背景

在检查AlphaPilot智能体发出的信号文件时，发现了两个关键问题：

1. **股票代码格式不匹配**
   - AlphaPilot格式：`603353.SH`、`300444.SZ`
   - 系统内部格式：`SHSE.603353`、`SZSE.300444`
   - 原代码使用简单分割导致匹配失败

2. **量比数据源混淆风险**
   - 曾错误尝试从掘金SDK获取量比（不支持）
   - 可能误用价格对比替代量比判断

```python
# 修复前：简单分割（错误）
sig_pure = sig_code.split('.')[-1]  # 603353.SH → SH (错误!)
```

## 🎓 经验教训总结

### 教训1：警惕平台API的局限性

**错误思维：**
> "掘金SDK没有量比字段，我自己算一个吧"

**正确思维：**
> "量比是AlphaPilot的核心决策指标，必须从信号文件读取。如果平台不支持，那是平台的问题，不是大脑的问题。"

---

### 教训2：区分"大脑逻辑"和"手脚适配"

**大脑逻辑（不可变）：**
- ✅ 量比计算算法
- ✅ 信号分流规则
- ✅ 止损止盈阈值
- ✅ 延时策略观察期

**手脚适配（可变）：**
- ✅ 股票代码格式转换
- ✅ 下单API调用方式
- ✅ 持仓查询接口
- ✅ 日志输出格式

---

### 教训3：标准化是关键

**最佳实践：**
```python
# 提取纯数字代码作为唯一标识
def normalize_code(code):
    if '.' in code:
        parts = code.split('.')
        # 返回数字部分
        return parts[0] if parts[0].isdigit() else parts[1]
    return code

# 统一比对基准
if normalize_code(sig_code) == normalize_code(system_code):
    match!
```
