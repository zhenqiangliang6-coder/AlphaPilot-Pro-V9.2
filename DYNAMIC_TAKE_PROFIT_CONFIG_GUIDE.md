## ⚙️ 可配置参数清单

### 1. 时间控制参数

```python
TAKE_PROFIT_EARLIEST_TIME = "0951"  # 最早执行时间（HHMM格式）
```

**说明**: 
- 默认 `0951` (09:51),避开开盘前剧烈波动
- **可调范围**: `"0930"` - `"1500"`
- **建议值**:
  - `"0930"` - 开盘立即执行(激进)
  - `"0951"` - 默认值(平衡)
  - `"1000"` - 更晚执行(保守)

---

### 2. 第一级止盈参数(快速止盈)

适用于**所有股票**,上涨后回落立即卖出:

```python
TAKE_PROFIT_LEVEL1_GAIN = 0.03      # 上涨阈值（3%）
TAKE_PROFIT_LEVEL1_DROP = 0.013     # 回落阈值（1.3%）
TAKE_PROFIT_LEVEL1_MAX = 0.085      # 涨幅上限（8.5%）
```

**工作原理**:
- 当股票上涨 ≥ 3% 且 ≤ 8.5% 时开始监控
- 如果从最高点回落 ≥ 1.3%,立即卖出
- **超过8.5%** 则不执行第一级,交由第二/三级处理

**调整建议**:
| 场景 | LEVEL1_GAIN | LEVEL1_DROP | LEVEL1_MAX | 说明 |
|------|------------|-------------|-----------|------|
| **激进** | 0.02 (2%) | 0.01 (1%) | 0.06 (6%) | 快速获利了结 |
| **平衡** | 0.03 (3%) | 0.013 (1.3%) | 0.085 (8.5%) | 默认值 |
| **保守** | 0.05 (5%) | 0.02 (2%) | 0.12 (12%) | 让利润奔跑 |

---

### 3. 第二级止盈参数(波段止盈)

适用于 **60/00开头的主板股票**:

```python
TAKE_PROFIT_LEVEL2_GAIN = 0.09          # 上涨阈值（9%）
TAKE_PROFIT_LEVEL2_HOLD_MINUTES = 12    # 持有时间（12分钟）
```

**工作原理**:
- 当主板股票上涨 ≥ 9% 时开始计时
- 持有满12分钟后,无论是否回落都卖出
- 适合捕捉主板股票的波段行情

**调整建议**:
| 场景 | LEVEL2_GAIN | HOLD_MINUTES | 说明 |
|------|------------|--------------|------|
| **短线** | 0.07 (7%) | 8 | 快速波段 |
| **平衡** | 0.09 (9%) | 12 | 默认值 |
| **中长线** | 0.12 (12%) | 20 | 延长持有 |

---

### 4. 第三级止盈参数(强势股止盈)

适用于 **68/30开头的科创板/创业板股票**:

```python
TAKE_PROFIT_LEVEL3_GAIN = 0.18          # 上涨阈值（18%）
TAKE_PROFIT_LEVEL3_HOLD_MINUTES = 12    # 持有时间（12分钟）
```

**工作原理**:
- 当科创板/创业板股票上涨 ≥ 18% 时开始计时
- 持有满12分钟后卖出
- 适合捕捉强势股的爆发行情

**调整建议**:
| 场景 | LEVEL3_GAIN | HOLD_MINUTES | 说明 |
|------|------------|--------------|------|
| **保守** | 0.15 (15%) | 10 | 提前锁定利润 |
| **平衡** | 0.18 (18%) | 12 | 默认值 |
| **激进** | 0.25 (25%) | 20 | 追求更高收益 |

## 🔧 如何调整参数

### 方法1: 直接编辑 settings.py

打开 [`config/settings.py`](file://d:\main_data\config\settings.py),找到对应参数修改:

```python
# 例如：想让第一级止盈更激进
TAKE_PROFIT_LEVEL1_GAIN = 0.02      # 从3%改为2%
TAKE_PROFIT_LEVEL1_DROP = 0.01      # 从1.3%改为1%
```

保存后重启策略即可生效。

---

### 方法2: 通过环境变量覆盖(高级)

在 `.env` 文件中添加:

```bash
TAKE_PROFIT_LEVEL1_GAIN=0.02
TAKE_PROFIT_LEVEL1_DROP=0.01
```

然后在 `settings.py` 中读取:

```python
TAKE_PROFIT_LEVEL1_GAIN = float(os.getenv('TAKE_PROFIT_LEVEL1_GAIN', '0.03'))
```

## 💡 调优建议

### 1. 先观察再调整

运行策略至少**1周**,观察日志中的止盈触发情况:

```
[止盈分析] SHSE.600000 成本:10.00 现价:10.50 盈亏:5.00%
[止盈-快速] SHSE.600000 未达到第一级阈值 (涨幅: 5.00%)

[止盈-快速] SZSE.300001 触发第一级止盈 (涨幅: 4.20%, 回落: 1.5%)
[TraderEngine] ✅ 下单成功: SELL SZSE.300001 ...
```

统计:
- 第一级触发频率: 多少次/天?
- 平均盈利: 每次止盈赚多少?
- 是否有"卖飞"现象?(卖出后继续大涨)

---

### 2. 逐步微调

**不要一次性大幅调整参数**,建议每次调整幅度不超过 **20%**:

```python
# ❌ 错误做法: 一次性从3%改到1%
TAKE_PROFIT_LEVEL1_GAIN = 0.01  # 变化太大!

# ✅ 正确做法: 逐步调整
# 第1步: 3% → 2.5%
TAKE_PROFIT_LEVEL1_GAIN = 0.025

# 运行3天观察效果

# 第2步: 2.5% → 2%
TAKE_PROFIT_LEVEL1_GAIN = 0.02
```

---

### 3. 分时段测试

不同时段可能需要不同参数:

| 时段 | 建议参数 | 原因 |
|------|---------|------|
| **早盘(09:30-10:30)** | 更宽松(阈值提高) | 波动大,容易误触发 |
| **上午(10:30-11:30)** | 默认值 | 市场稳定 |
| **下午(13:00-14:00)** | 更严格(阈值降低) | 临近收盘,快速锁定利润 |
| **尾盘(14:00-15:00)** | 最严格 | 避免隔夜风险 |

⚠️ **注意**: 当前版本不支持分时段动态调整,未来版本可能会加入此功能。

## 🎯 常见问题

### Q1: 为什么我的股票没有触发止盈?

**检查清单**:
1. 当前时间是否在 `TAKE_PROFIT_EARLIEST_TIME` 之后?
2. 涨幅是否达到对应级别的阈值?
3. 如果是第一级,是否已经回落足够幅度?
4. 查看日志中是否有 `[止盈分析]` 输出?

---

### Q2: 止盈触发太频繁怎么办?

**解决方案**:
- 提高 `LEVEL1_GAIN`(例如从3%提高到5%)
- 提高 `LEVEL1_DROP`(例如从1.3%提高到2%)
- 或延迟执行时间 `EARLIEST_TIME`(例如从09:51改为10:00)

---

### Q3: 止盈触发太少,经常"卖飞"怎么办?

**解决方案**:
- 降低 `LEVEL1_GAIN`(例如从3%降到2%)
- 降低 `LEVEL1_DROP`(例如从1.3%降到1%)
- 提高 `LEVEL1_MAX`(例如从8.5%提高到12%)

---

### Q4: 如何禁用某一级止盈?

**方法**: 将阈值设置为极大值

```python
# 禁用第一级止盈
TAKE_PROFIT_LEVEL1_GAIN = 0.99  # 几乎不可能达到

# 禁用第二级止盈
TAKE_PROFIT_LEVEL2_GAIN = 0.99

# 禁用第三级止盈
TAKE_PROFIT_LEVEL3_GAIN = 0.99
```
