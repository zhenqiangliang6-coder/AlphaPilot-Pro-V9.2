## 🔍 **根本原因**

### **掘金API的字段名差异**

掘金量化平台返回的持仓对象中，成本价字段名为 **`cost_price`**，但代码中统一使用 **`open_price`**，导致：

1. `getattr(pos, 'open_price', 0.0)` → 找不到 → 返回默认值 `0.0`
2. 所有模块判断 `if open_price <= 0` → 跳过
3. **止损、止盈、火箭加仓全部失效！**

### **实际持仓数据（从截图）**

**但这些数据无法被策略读取！**

## ✅ **修复方案**

### **统一兼容模式**

所有模块现在使用相同的字段获取逻辑：

```python
# 【关键修复】兼容多种成本价字段名
# 掘金API可能返回: open_price, cost_price, avg_price
open_price = getattr(pos, 'open_price', 0.0)
if open_price <= 0:
    open_price = getattr(pos, 'cost_price', 0.0)  # ← 掘金字段名
if open_price <= 0:
    open_price = getattr(pos, 'avg_price', 0.0)   # ← 备用字段名

if open_price <= 0:
    log.log("[跳过] {} 成本价为0，跳过".format(code))
    continue
```
