## 🔍 **详细分析**

### **动态止盈模块的三级策略**

#### **第一级：快速止盈（所有股票）**
- **触发条件**：上涨 ≥ 3% 且 ≤ 65%，然后回落 ≥ 1.3%
- **执行动作**：立即卖出全部持仓
- **适用场景**：短线快速获利了结

#### **第二级：波段止盈（60/00开头股票）**
- **触发条件**：上涨 ≥ 9%，持有 ≥ 12 分钟
- **执行动作**：卖出全部持仓
- **适用场景**：主板股票波段操作

#### **第三级：强势股止盈（68/30开头股票）**
- **触发条件**：上涨 ≥ 18%，持有 ≥ 12 分钟
- **执行动作**：卖出全部持仓
- **适用场景**：科创板/创业板强势股

---

### **时间控制机制**

```python
# 动态止盈最早执行时间（HHMM格式）
self.EARLIEST_EXECUTION_TIME = "0951"  # 默认 09:51

# 双重时间过滤：
# 1. heartbeat.py 中：if now_time >= "0951"
# 2. dynamic_take_profit.py 中：if current_time < self.EARLIEST_EXECUTION_TIME
```

**设计理由**：
- 避开开盘前5分钟的剧烈波动（09:30-09:35）
- 给市场足够的稳定时间，避免误触发
- 可根据需要调整为 `"0930"`（开盘即执行）或 `"1000"`（更晚执行）

**如需调整**：
```python

# 或
self.EARLIEST_EXECUTION_TIME = "1000"  # 改为10点执行
```

同时修改 [heartbeat.py](file://d:\mpython\utils\heartbeat.py) 第 90 行：

### **3. 止盈与止损的优先级**

当前系统中：
- **止损**：每15秒检查一次，无时间限制（全天执行）
- **止盈**：每15秒检查一次，09:51之后执行

**注意**：止损和止盈是**独立并行**的，互不影响。如果一只股票同时触发止损和止盈，**先执行的会生效**。
