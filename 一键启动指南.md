## 🔧 故障排查

### 问题1: "未找到Python"

**解决:**
1. 下载并安装 Python 3.8+
2. 安装时勾选 "Add Python to PATH"
3. 重启命令行后重试

### 问题2: "源目录中没有信号文件"

**解决:**
1. 确认 `D:\mpython\signals\processed` 目录存在
2. 确认该目录下有 `.txt` 格式的信号文件
3. 如果目录不同,修改脚本中的路径配置

### 问题3: 依赖安装失败

**解决:**
```bash
# 手动安装依赖
pip install gm python-dotenv watchdog -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题4: 策略没有响应

**检查:**
1. 掘金终端是否已登录
2. 仿真账户是否正常
3. 查看 `logs` 目录下的日志文件

## 💡 使用技巧

### 技巧1: 只读观察模式

在正式实盘前,先开启只读模式避免误下单:

编辑 `config/settings.py`:
```python
READ_ONLY_MODE = True  # 只记录日志,不下单
TEST_SAFE_MODE = True  # 限制下单金额
```

### 技巧2: 中断后继续

如果中途按 `Ctrl+C` 中断:
- ✅ 已同步的文件会被跳过
- ✅ 下次运行从未同步的位置继续
- ✅ 不会重复处理

### 技巧3: 多轮测试

**第一轮:** 快速验证逻辑

```python
DELAY_SECONDS = 2
MAX_FILES = 10
```

**第二轮:** 完整回测
```python

MAX_FILES = 0
```

**第三轮:** 多账户对比
```bash

```
