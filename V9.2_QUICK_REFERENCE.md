# V9.2 策略优化 - 快速参考指南

## 🎯 核心改动一览

### ✅ 功能1: 分时段量比阈值

| 时间段 | 新阈值 | 旧阈值 | 变化 |
|--------|--------|--------|------|
| 09:30-10:30 | **1.2** | 2.0 | ⬇️ 降低40% |
| 10:30-11:30 | **2.2** | 2.0 | ⬆️ 提高10% |
| 13:00-14:00 | **3.0** | 3.6 | ⬇️ 降低17% |
| 14:00-15:00 | **3.8** | 3.6 | ⬆️ 提高6% |

**弱势市场** (大盘-1.0%至-0.35%): 所有阈值 × 1.5

---

### ✅ 功能2: 33%仓位控制

| 阶段 | 仓位比例 | 触发条件 | 累计仓位 |
|------|---------|---------|---------|
| 首次买入 | **33%** | 信号触发 | 33% |
| 一级火箭 | **+33%** | 总浮盈≥3万 | 66% |
| 二级火箭 | **+33%** | 总浮盈≥7万 | 99% |

**安全机制**:
- 每次下单 ≤ 可用现金的80%
- 单笔上限 50,000元
- 最小金额 15,000元

## 🔧 如何调整参数

### 修改量比阈值

编辑 `strategies/signal_strategy.py` 第270行左右:

```python
vr_thresholds = {
    "morning_early": 1.2,    # 改这里
    "morning_late": 2.2,     # 改这里
    "afternoon_early": 3.0,  # 改这里
    "afternoon_late": 3.8,   # 改这里
}
```

### 修改仓位比例

编辑 `config/settings.py`:

```python
INITIAL_POSITION_RATIO = 0.33   # 改这里 (0.33 = 33%)
BOOST_POSITION_RATIO = 0.33     # 改这里
FIXED_ORDER_AMOUNT = 50000.0    # 改这里 (单笔上限)
```

### 修改火箭加仓阈值

编辑 `config/settings.py`:

```python
LEVEL_1_THRESHOLD = 30000.0     # 改这里 (一级火箭)
LEVEL_2_THRESHOLD = 70000.0    # 改这里 (二级火箭)
```

## ⚠️ 重要提醒

### 1. 测试建议

✅ **第一步**: 在掘金仿真账户运行至少1周  
✅ **第二步**: 统计各时段的信号通过率  
✅ **第三步**: 根据实际效果微调参数  
❌ **禁止**: 直接在实盘账户使用新参数

### 2. 观察要点

- **早盘**: 是否捕获更多有效信号?
- **上午**: 是否过滤掉假突破?
- **仓位**: 首次33%是否足够?是否需要调整?
- **加仓**: 火箭触发频率是否合理?

### 3. 常见问题

**Q: 为什么早盘量比降低反而更安全?**  
A: 早盘波动大,降低阈值可以捕捉真正的强势股,同时大盘联动和弱势市场机制会过滤风险。

**Q: 33%仓位会不会太少?**  
A: 这是经典的"三分之一法则",保留现金应对波动。如果盈利可以通过火箭加仓扩大仓位。

**Q: 如何知道当前使用的是哪个时段的阈值?**  
A: 查看日志中的时段标识,如 `[买入通过] 早盘(09:30-10:30)`。
