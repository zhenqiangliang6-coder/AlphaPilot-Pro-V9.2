## 🎯 正确的工作流程

### 第一步：在掘金终端中创建策略实例

1. 打开掘金量化终端
2. 点击顶部菜单 **"量化研究"** → **"新建策略"**
3. 选择策略类型（Python策略）
4. 填写策略名称（如 "AlphaPilot Pro"）
5. **系统自动生成 `strategy_id`**（如 `a62d366d-3c78-11f1-8563-1ece51d839d6`）
6. 记下这个ID（在策略设置中查看）

### 第二步：获取策略ID和目录路径

1. 在掘金IDE中找到刚创建的策略
2. 查看策略属性，找到：
   - **策略ID**: `a62d366d-3c78-11f1-8563-1ece51d839d6`
   - **策略目录**: `D:\mpython\a62d366d-3c78-11f1-8563-1ece51d839d6\`

### 第三步：在VSCode中配置代码

打开项目根目录的 [main.py](file://d:\mpython\main.py)，修改 [run()](file://d:\mpython\main.py#L189-L196) 函数：

```python
run(
    strategy_id='a62d366d-3c78-11f1-8563-1ece51d839d6',  # ← 使用终端的ID
    filename='main.py',
    mode=MODE_LIVE,
    token='fdf08e9d00c4da3b635c2616724ddae3f7793562'
)
```

### 第四步：同步代码到策略目录

运行同步脚本：

```powershell
cd D:\mpython
.\sync_to_gm_ide.ps1
```

### 第五步：在掘金IDE中运行

1. 打开掘金IDE中的策略文件
2. 点击"运行"按钮 (▶️)
3. 策略开始执行

### 常见错误日志及处理

#### ❌ 错误1: 无效的ACCOUNT_ID
```
❌ 下单失败: {"status": 1020, "message": "无效的ACCOUNT_ID", ...}
```

**原因**: 掘金终端中账户未连接或多账户冲突

**解决**:
1. 进入"账户管理"
2. 断开所有账户
3. 重新连接目标账户
4. 确认状态为"已连接"
5. 重新运行策略

#### ❌ 错误2: ModuleNotFoundError
```
ModuleNotFoundError: No module named 'xxx'
```

**原因**: Python解释器环境问题或文件未同步

**解决**:
1. 运行同步脚本: `.\sync_to_gm_ide.ps1`
2. 清理缓存: `Remove-Item -Recurse -Force __pycache__`
3. 检查掘金IDE的Python解释器配置

#### ❌ 错误3: 连接失败
```
ERROR [callback.py:xxx] 连接交易服务失败
```

**原因**: 网络问题或Token失效

**解决**:
1. 检查网络连接
2. 重新设置Token
3. 重启掘金终端

## 🛠️ 常见问题解决

### Q1: 运行时报错 `strategy_id` 不匹配？

**症状**：
```
Error: strategy_id mismatch
```

**原因**：代码中的 [strategy_id](file://d:\mpython\config\settings.py#L30-L30) 与掘金终端中的策略实例ID不一致

**解决**：
1. 在掘金终端中查看策略属性，复制正确的 [strategy_id](file://d:\mpython\config\settings.py#L30-L30)
2. 修改 [main.py](file://d:\mpython\main.py) 中的 [run()](file://d:\mpython\main.py#L189-L196) 函数
3. 重新同步并运行

### Q2: ModuleNotFoundError？

**解决**：
1. 运行同步脚本: `.\sync_to_gm_ide.ps1`
2. 检查掘金IDE的Python解释器配置
3. 确认依赖已安装

### Q3: 无效的ACCOUNT_ID？

**解决**：
1. 确认在掘金终端"账户管理"中账户已连接
2. 建议只连接一个账户，避免冲突
3. 重新运行策略
