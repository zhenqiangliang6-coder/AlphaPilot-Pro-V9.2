# 30/68差异化止盈止损功能 - 快速参考指南

**版本**: V9.2.2  
**更新日期**: 2026-05-14  

---

## 🎯 一、功能概述

为科创板（68开头）和创业板（30开头）股票提供专属的止盈止损策略，充分利用其高波动特性。

---

## 📊 二、策略对比

| 参数 | 普通股票（60/00） | 特殊股票（30/68） | 说明 |
|------|------------------|------------------|------|
| **一级止损** | -1.2% 减半 | -1.6% 减半 | +0.4%宽松度 |
| **二级止损** | -2.5% 清仓 | -3.5% 清仓 | +1.0%宽松度 |
| **止盈回落** | 1.3% | 1.5% | +0.2%容忍度 |
| **止盈上限** | 8.5% | 17.0% | +8.5%空间 |

---

## ⚙️ 三、配置参数

### 3.1 止损配置（`config/settings.py`）

```python
# 普通股票
STOP_LOSS_LEVEL1_THRESHOLD = 0.012      # -1.2%
STOP_LOSS_LEVEL2_THRESHOLD = 0.025      # -2.5%

# 30/68股票
STOP_LOSS_LEVEL1_THRESHOLD_3068 = 0.016     # -1.6%
STOP_LOSS_LEVEL2_THRESHOLD_3068 = 0.035     # -3.5%
```

### 3.2 止盈配置（`config/settings.py`）

```python
# 普通股票
TAKE_PROFIT_LEVEL1_DROP = 0.013         # 回落1.3%
TAKE_PROFIT_LEVEL1_MAX = 0.085          # 上限8.5%

# 30/68股票
TAKE_PROFIT_LEVEL1_DROP_3068 = 0.015    # 回落1.5%
TAKE_PROFIT_LEVEL1_MAX_3068 = 0.17      # 上限17%
```

## ⚠️ 六、注意事项

### 6.1 T+1合规
- ✅ 所有卖出操作基于 `can_use_volume`（可卖数量）
- ✅ 当日买入不可当日卖出
- ✅ 卖出数量自动取整为100股整数倍

### 6.2 风险控制
- 建议降低30/68股票的初始仓位（如从10%降至7%）
- 密切观察止损触发频率
- 记录实际盈亏比

- 掘金标准：`SZSE.300444`、`SHSE.688295`

## 📞 七、问题排查

### 7.1 股票类型识别错误

**现象**：日志显示"普通股票"但实际是30/68开头

**排查**：
```bash
python test_3068_differentiated_strategy.py
```

检查"代码前缀识别测试"部分是否全部通过。

### 7.2 参数未生效

**现象**：30/68股票仍使用-1.2%止损

**排查**：
1. 检查 `config/settings.py` 是否包含 `_3068` 后缀的参数
2. 清理缓存：`Get-ChildItem -Path . -Include __pycache__ -Recurse | Remove-Item -Recurse -Force`
3. 重启程序

### 7.3 止盈未触发

**现象**：30/68股票涨到15%仍未止盈

**原因**：第一级止盈需要"上涨后回落"，不是达到上限就卖出

**逻辑**：
- 最高涨到12%，当前回落到10.3%（回落1.7% > 1.5%阈值）→ 触发
- 持续上涨到16%，未回落 → 不触发，等待第二/三级止盈
