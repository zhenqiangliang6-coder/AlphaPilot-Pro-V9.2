## ✨ 新增功能

### 功能1: 分时段差异化量比阈值 ⭐⭐⭐

#### 🎯 设计理念

不同交易时段的市場特征不同,需要差异化的量比过滤标准:
- **早盘**: 市场刚开盘,波动较大,适当降低量比要求捕捉机会
- **上午中段**: 市场趋于稳定,提高量比要求过滤噪音
- **下午**: 临近收盘,需要更强的量能确认
- **尾盘**: 最后交易时段,最严格的量比要求避免追高

#### 📋 具体规则

| 时间段 | 量比阈值 | 弱势市场阈值 | 说明 |
|--------|---------|------------|------|
| **09:30-10:30** | ≥ 1.2 | ≥ 1.8 | 早盘宽松,快速反应 |
| **10:30-11:30** | ≥ 2.2 | ≥ 3.3 | 上午严格,过滤假突破 |
| **13:00-14:00** | ≥ 3.0 | ≥ 4.5 | 下午更严格,确认趋势 |
| **14:00-15:00** | ≥ 3.8 | ≥ 5.7 | 尾盘最严格,避免追高 |

**弱势市场定义**: 上证指数涨跌幅在 -1.0% 至 -0.35% 之间  
**弱势市场调整**: 所有时段量比阈值 × 1.5

#### 💡 示例场景

```python
# 场景1: 早盘09:45,大盘涨0.5%,某股票量比1.5
✅ 通过: 1.5 >= 1.2 (早盘阈值)

# 场景2: 上午11:00,大盘涨0.5%,某股票量比1.8
❌ 拦截: 1.8 < 2.2 (上午阈值)

# 场景3: 下午13:30,大盘跌-0.8%(弱势),某股票量比4.0
✅ 通过: 4.0 >= 4.5? No... 等等,4.0 < 4.5
❌ 拦截: 弱势市场需要 3.0 * 1.5 = 4.5

# 场景4: 尾盘14:30,大盘涨1.0%,某股票量比4.2
✅ 通过: 4.2 >= 3.8 (尾盘阈值)
```

### 功能2: 精细化仓位控制 ⭐⭐⭐

#### 🎯 设计理念

采用**金字塔式建仓**策略,严格控制风险:
1. **首次买入**: 使用总资产的33%,保留充足现金应对波动
2. **火箭加仓**: 必须利润达标(浮盈≥3万/7万)才能追加33%
3. **最大仓位**: 通过多次加仓累积,理论上可达99%(3次×33%)

#### 📋 仓位分配规则

| 阶段 | 触发条件 | 仓位比例 | 累计仓位 | 说明 |
|------|---------|---------|---------|------|
| **首次买入** | 信号触发 | 33% | 33% | 基础仓位,谨慎入场 |
| **一级火箭** | 总浮盈≥3万 | +33% | 66% | 盈利确认后加仓 |
| **二级火箭** | 总浮盈≥7万 | +33% | 99% | 强势行情最大化 |

#### 💡 示例场景

假设总资产100万元:

```python
# 初始状态
总资产: 1,000,000元
可用现金: 1,000,000元

# 第一次买入(信号触发)
买入金额: 1,000,000 × 33% = 330,000元
剩余现金: 670,000元
当前仓位: 33%

# 持仓上涨,总浮盈达到30,000元
触发一级火箭加仓
加仓金额: 1,000,000 × 33% = 330,000元
剩余现金: 340,000元
当前仓位: 66%

# 继续上涨,总浮盈达到70,000元
触发二级火箭加仓
加仓金额: 1,000,000 × 33% = 330,000元
剩余现金: 10,000元
当前仓位: 99%
```

#### 🔒 安全机制

1. **现金保留**: 每次下单不超过可用现金的80%,保留安全边际
2. **单笔上限**: 单次买入不超过50,000元(可配置)
3. **最小金额**: 低于15,000元的计算结果会被拦截
4. **利润验证**: 火箭加仓必须真实盈利,防止亏损加仓

## 🎯 策略优势

✅ **适应市场节奏**: 早盘宽松捕捉机会,尾盘严格避免追高

✅ **盈利验证**: 火箭加仓必须真实盈利,防止亏损加码  
✅ **灵活扩展**: 理论上可通过3次加仓达到99%仓位  
✅ **安全边际**: 每次下单保留20%现金,应对突发情况

## ⚠️ 注意事项

### 1. 分时段量比阈值

- **时间判断**: 基于系统本地时间,确保服务器时间准确
- **非交易时段**: 不在4个时段内的信号会被拦截
- **弱势市场**: 大盘-1.0%至-0.35%时阈值自动提高50%
- **日志观察**: 初期建议密切观察各时段的通过率

### 2. 仓位控制

- **总资产计算**: 基于`total_asset`而非`cash`,包含持仓市值
- **现金不足**: 如果可用现金不足,实际下单金额会降低
- **火箭加仓**: 必须总浮盈达到阈值,不是单只股票盈利
- **重复保护**: 同一股票9分钟内不会重复下单

### 3. 测试建议

1. **仿真账户测试**: 先在掘金仿真账户验证逻辑
2. **观察日志**: 重点关注时段切换时的阈值变化
3. **仓位跟踪**: 记录每次买入的金额和比例
4. **参数调优**: 根据实际效果微调量比阈值

## 💡 量化专家建议

### 1. 关于分时段量比

> "不同时段的市场微观结构不同。早盘流动性较低但波动大,适合降低量比要求;尾盘流动性高但容易追高,需要更严格的过滤。这种设计符合A股市场的实际特征。"

### 2. 关于仓位控制

> "33%初始仓位是经典的'三分之一法则',既保证有足够 exposure 获取收益,又保留充足现金应对波动。金字塔式加仓确保只有在盈利时才扩大风险暴露,这是专业基金管理的常用策略。"

### 3. 风险提示

> "任何策略都需要实盘验证。建议先在仿真账户运行至少2周,统计各时段的胜率和盈亏比,再根据实际情况微调参数。记住,没有完美的参数,只有适合您风险偏好的参数。"
