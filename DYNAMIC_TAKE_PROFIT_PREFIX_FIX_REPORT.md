# 动态止盈模块股票代码前缀BUG修复报告

## 📋 问题概述

**发现时间**: 2026-05-07  
**问题类型**: 逻辑错误 - 股票代码格式不匹配  
**影响范围**: 动态止盈模块（Level 2 和 Level 3）  
**严重程度**: 🔴 **高** - 导致大部分止盈策略无法执行

---

## 🔍 问题根因分析

### 1. 掘金SDK返回的股票代码格式

掘金量化平台 v3.0.183 返回的持仓数据中，`symbol` 字段格式为：
- **深圳股票**: `SZSE.300444`、`SZSE.000001`
- **上海股票**: `SHSE.688295`、`SHSE.600519`

### 2. 原代码的错误逻辑

在 `risk/dynamic_take_profit.py` 中：

```python
# ❌ 错误代码（第237行和第307行）
def _check_level2(self, code, ...):
    code_prefix = code[:2]  # 提取前两位字符
    if code_prefix not in ['60', '00']:
        return
```

**问题分析**：
- 对于 `SZSE.300444`，`code[:2]` 得到的是 `"SZ"`，而不是期望的 `"30"`
- 对于 `SHSE.688295`，`code[:2]` 得到的是 `"SH"`，而不是期望的 `"68"`
- 结果：**所有 Level 2 和 Level 3 止盈条件永远无法满足**

### 3. 实际影响

| 止盈级别 | 适用股票 | 原逻辑判断结果 | 实际执行情况 |
|---------|---------|--------------|------------|
| Level 1 (快速) | 所有股票 | ✅ 不检查前缀 | **正常执行** |
| Level 2 (波段) | 60/00开头 | ❌ 前缀为"SZ"/"SH" | **永不执行** |
| Level 3 (强势) | 68/30开头 | ❌ 前缀为"SZ"/"SH" | **永不执行** |

**用户反馈验证**：
> "我总发觉止盈好像没有执行的？？"

这正是由于 Level 2 和 Level 3 完全失效导致的。

## ✅ 修复方案

### 核心思路

参考项目中已有的 `core/state_manager.py` 中的 `_normalize_stock_code()` 函数，采用**先提取纯数字代码，再判断前缀**的策略。

### 具体修改

#### 1. 新增辅助方法 `_extract_numeric_code()`

```python
def _extract_numeric_code(self, code):
    """
    【V9.2.1 新增】从掘金格式代码中提取纯数字部分
    
    参数：
        code: 股票代码，可能是以下格式：
            - SZSE.300444 (掘金标准格式)
            - SHSE.688295 (掘金标准格式)
            - 300444.SZ (外部信号格式)
            - 300444 (纯数字)
    
    返回：
        str: 纯数字股票代码，如 "300444"
    """
    if not code:
        return ""
    
    # 如果包含点号，分割处理
    if '.' in code:
        parts = code.split('.')
        # 尝试找到纯数字部分
        for part in parts:
            if part.isdigit():
                return part
        # 如果没有纯数字部分，返回第一部分
        return parts[0]
    
    # 纯数字代码，直接返回
    if code.isdigit():
        return code
    
    # 无法识别，返回原样
    return code
```

#### 2. 修改 `_check_level2()` 方法


## ⚠️ 注意事项

### 1. 时间窗口限制

动态止盈默认在 **09:51** 之后才执行，避免开盘剧烈波动导致误触发。

- 如需调整，修改 `settings.py` 中的 `TAKE_PROFIT_EARLIEST_TIME` 参数
- 或在 `dynamic_take_profit.py` 中修改 `EARLIEST_EXECUTION_TIME` 常量

### 2. T+1 规则

根据A股交易规则，**今日买入不可今日卖出**。止盈模块已实现该检查：

```python
if can_sell <= 0:
    log.log("[止盈跳过] {} 今日买入不可卖（总持仓:{} 可卖:0），无法执行".format(code, total_volume))
    return
```

### 3. 卖出数量规范

A股要求卖出数量必须是 **100股的整数倍**：

```python
actual_volume = (volume // 100) * 100
```

### 4. 检查频率

止盈检查由心跳线程每 **15秒** 执行一次（配置在 `utils/heartbeat.py` 中）。
