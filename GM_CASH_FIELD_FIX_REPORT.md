## 📋 问题描述

### 现象1：初始错误
日志显示：
```
💰 [账户资金]
   可用资金: ¥0.00        ❌ 错误
   总资产:   ¥869,458.99  ❌ 错误（实际是持仓市值）
   持仓市值: ¥869,458.99  ✅ 正确
```

但实际掘金终端显示：
- **总资产**: 987,674.02 元
- **可用资金**: 118,215.03 元
- **持仓市值**: 869,458.99 元

### 现象2：字段理解错误
第一次修复后，日志显示：
```
💰 [账户资金]
   可用资金: ¥987,674.02   ❌ 错误（这是总资产！）
   总资产:   ¥1,857,133.02 ❌ 错误（重复计算了）
   持仓市值: ¥869,458.99   ✅ 正确
```

**问题**: 误将 `nav` 字段当作可用资金，实际上它是**总资产**！

---

## 🔍 根本原因

### SDK字段真实含义

掘金量化SDK v3.0.183 的 `get_cash()` 返回的Cash对象中：

| 字段名 | 真实含义 | 数值示例 |
|--------|---------|---------|
| **`nav`** | **总资产** (Net Asset Value) | 987,674.02 |
| 可用资金 | **需要计算** = nav - 持仓市值 | 118,215.03 |
| 持仓市值 | 需要从持仓查询累加 | 869,458.99 |

### 错误理解链

1. **第一次错误**：假设 `cash` 字段存在 → 返回0
2. **第二次错误**：假设 `nav` 是可用资金 → 返回总资产
3. **正确理解**：`nav` = 总资产，可用资金 = nav - market_value

## ✅ 最终修复方案

### 正确的字段映射逻辑

在 [`core/trader_engine.py`](d:\mpython\core\trader_engine.py) 的 `query_asset()` 方法中：

```python
# 1. 获取总资产（来自 nav 字段）
total_asset = 0.0
if hasattr(cash_info, 'nav'):
    total_asset = cash_info.nav  # ✅ nav 是总资产

# 2. 查询持仓市值
positions = self.query_positions()
market_value = sum(pos.market_value for pos in positions) if positions else 0

# 3. 计算可用资金 = 总资产 - 持仓市值
available_cash = total_asset - market_value

# 4. 组装返回数据
asset_info = {
    'cash': available_cash,        # 可用资金 = 总资产 - 持仓市值
    'total_asset': total_asset,    # 总资产（来自 nav）
    'market_value': market_value,  # 持仓市值
}
```

### 修复后的正确输出

```

   总资产:   ¥987,674.02  ✅ 正确

```

## 🎯 经验总结

### 1. SDK字段名与含义的双重验证

**不要仅假设字段名，还要验证字段含义**！

| 步骤 | 操作 | 目的 |
|------|------|------|
| 1️⃣ | 使用 `dir(obj)` 查看字段名 | 确认有哪些字段 |
| 2️⃣ | 打印字段值并与GUI对比 | 确认字段的真实含义 |
| 3️⃣ | 验证计算逻辑 | 确保字段间关系正确 |

### 2. 掘金SDK资金字段正确理解

| 概念 | 字段名 | 获取方式 | 备注 |
|------|--------|---------|------|
| 总资产 | `nav` | `cash_info.nav` | Net Asset Value |
| 持仓市值 | - | 查询持仓累加 | `sum(pos.market_value)` |
| 可用资金 | - | **计算得出** | `nav - market_value` |

### 3. 调试技巧升级版

当遇到字段含义不确定时：

```python
# 步骤1: 打印所有字段及其值
cash_info = get_cash()
for attr in dir(cash_info):
    if not attr.startswith('_'):
        print(f"{attr}: {getattr(cash_info, attr)}")

# 步骤2: 与掘金终端GUI对比
# 步骤3: 验证字段间的数学关系
# 总资产 = 可用资金 + 持仓市值 + 冻结资金
```
