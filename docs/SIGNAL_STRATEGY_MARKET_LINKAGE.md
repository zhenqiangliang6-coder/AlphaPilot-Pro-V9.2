## 1. 概述
本模块位于 `strategies/signal_strategy.py`，负责处理外部信号文件并执行即时买卖操作。为了降低市场系统性风险，V9.2 版本引入了**大盘联动控制机制**，即个股的交易决策不仅取决于其自身的量比（Volume Ratio），还严格受限于上证指数（SHSE.000001）的实时表现。

## 2. 核心逻辑说明

### 2.1 大盘基准定义
*   **参考指数**: 上证指数 (`SHSE.000001`)
*   **计算基准**: **当日开盘价** (Open Price)
*   **涨跌幅公式**: `(当前价 - 开盘价) / 开盘价 * 100%`
*   **数据获取**: 通过掘金量化 API `current(['SHSE.000001'])` 实时获取。

### 2.2 时段区分
策略根据一天中的不同时段采用不同的风控标准：
*   **上午时段**: 09:30 - 11:30
*   **下午时段**: 13:00 - 15:00

### 2.3 交易阈值矩阵

| 交易方向 | 大盘区间 (涨跌幅) | 上午量比要求 | 下午量比要求 | 逻辑说明 |
| :--- | :--- | :--- | :--- | :--- |
| **买入 (BUY)** | **-0.35% ~ +1.9%** | ≥ 2.0 | ≥ 3.6 | 正常震荡市，下午要求更高确认度 |
| **买入 (BUY)** | **-1.0% ~ -0.35%** | ≥ 3.6 | ≥ 5.0 | 弱势调整市，需极强资金介入才参与 |
| **卖出 (SELL)** | **-0.35% ~ +1.9%** | ≥ 1.5 | ≥ 1.5 | 正常流动性即可离场 |
| **卖出 (SELL)** | **-1.0% ~ -0.35%** | ≥ 0.8 | ≥ 0.8 | 弱势下只要有成交即允许止损 |
| **禁止交易** | **其他区间** | - | - | 极端行情或数据缺失时保护本金 |

## 3. 代码实现细节

### 3.1 关键方法 `_decide_action`
该方法在下单前被调用，返回 `True` 表示允许交易，`False` 表示拦截。

```python
def _decide_action(self, action, vr):
    """
    根据大盘涨跌幅、量比和时间段决定是否交易
    """
    index_change = self._get_index_change() # 获取上证指数涨跌幅
    
    # 1. 判断时段
    now = datetime.datetime.now()
    time_str = now.strftime("%H%M")
    is_morning = ("0930" <= time_str <= "1130")
    is_afternoon = ("1300" <= time_str <= "1500")

    # 2. 买入逻辑
    if action == "BUY":
        if -0.35 <= index_change <= 1.9:
            return (is_morning and vr >= 2.0) or (is_afternoon and vr >= 3.6)
        if -1.0 <= index_change < -0.35:
            return (is_morning and vr >= 3.6) or (is_afternoon and vr >= 5.0)
        return False # 超出安全区间

    # 3. 卖出逻辑
    else:
        if -0.35 <= index_change <= 1.9:
            return vr >= 1.5
        if -1.0 <= index_change < -0.35:
            return vr >= 0.8
        return False
```

### 3.2 仓位计算 `_check_position_and_calculate_volume`
*   **资金管理**: 单笔买入金额 = `可用资金 * 80%`。
*   **上限控制**: 单笔最大不超过 50,000 元。
*   **取整规则**: 自动向下取整至 100 股的整数倍。

## 4. 调试与监控

### 4.1 日志输出示例
当信号触发但被大盘逻辑拦截时，日志将显示：
```text
[解析] JSON 错误 signal_batch_xxx.txt: [立即策略-阻断] SHSE.603353 属于延时股票，禁止立即买入
[仓位] SZSE.300641 计算失败: 拒绝交易 (大盘涨跌幅 -1.2%, 量比 2.1 未达下午弱势阈值 5.0)
```

### 4.2 常见问题排查
1.  **无法获取指数数据**: 检查掘金终端是否正常运行，以及网络连接是否通畅。
2.  **下午不买入**: 确认当前量比是否达到下午的高阈值（如弱势时需 ≥ 5.0）。
3.  **Processed 文件静默**: 检查日志中是否有 `[归档] ... (包含有效信号)` 字样，若无则说明信号文件格式有误。

## 5. 维护建议
*   **阈值调优**: 若发现策略在震荡市中过于保守，可适当降低下午时段的量比阈值（如从 3.6 降至 3.0）。
*   **指数扩展**: 未来可考虑增加深证成指或创业板指的加权平均作为大盘参考，以适应不同风格的市场。
