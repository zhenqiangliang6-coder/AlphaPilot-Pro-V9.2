# AlphaPilot Pro V9.1 - 风控模块优化报告

## 📋 优化日期
2026-04-21

## 🎯 优化目标
根据用户反馈，对动态止损、动态止盈和持仓日志进行优化，确保：
1. 风控模块（止损/止盈）有清晰的日志和执行记录
2. 个股持仓日志精简，避免刷屏
3. 符合项目代码规范（使用 `.format()` 替代 f-string）

---

## 🔍 问题诊断

### **1. 动态止损模块**
✅ **原有功能完整**：
- 三级止损机制（监控 -0.8%、一级 -1.7%、二级 -3.5%）
- 反弹保护机制
- 时间窗口控制（10:45-14:50）

⚠️ **发现问题**：
- 日志使用 f-string，违反项目规范
- 心跳模块中检查频率过高（每5秒）

### **2. 动态止盈模块**
✅ **原有功能完整**：
- 三级止盈机制（快速回落、波段持有、强势持有）
- 时间窗口控制（09:51 之后）
- 按股票代码前缀分类（60/00、68/30）

⚠️ **发现问题**：
- 心跳模块中检查频率与模块内部时间检查不一致

### **3. 个股持仓日志**
❌ **过于详细**：
- 每次心跳（60秒）都会详细打印每只股票的持仓数量、可卖数量、成本价
- 持仓较多时造成日志刷屏，影响关键信息的查看

---

## 🛠️ 优化方案

### **1. 精简持仓日志** ✅
**文件**: [`main.py`](file://d:\mpython\main.py)

**修改前**：
```python
for pos in positions_raw:
    stock_code = pos.get('symbol', 'N/A')
    volume = pos.get('volume', 0)
    can_use = pos.get('available', pos.get('available_today', 0))
    cost_price = pos.get('vwap', 0)
    
    log.log(f"   {stock_code}:")
    log.log(f"     持仓数量: {volume} 股")
    log.log(f"     可卖数量: {can_use} 股")
    log.log(f"     成本价:   ¥{cost_price:.2f}")
```

**修改后**：
```python
# 【精简】只汇总显示持仓数量和盈亏概况，不逐个打印详情
total_stocks = len(positions_raw)
total_volume = sum(pos.get('volume', 0) for pos in positions_raw)

log.log("📊 [当前持仓] 共 {} 只股票，总计 {} 股".format(total_stocks, total_volume))
```

**效果**：
- 日志从原来的 4*N 行减少到 1 行（N 为持仓股票数量）
- 保留了关键信息（股票数量、总股数）
- 需要查看个股详情时可临时取消注释详细打印代码

---

### **2. 修复止损模块日志格式** ✅
**文件**: [`risk/stop_loss.py`](file://d:\mpython\risk\stop_loss.py)

**修改内容**：
将所有 f-string 日志改为 `.format()` 格式，例如：

```python
# 修改前
log.log(f"[止损-监控] {code} 进入监控状态 (成本:{open_price:.2f} 现价:{current_price:.2f} 亏损:{loss_ratio*100:.2f}%)")

# 修改后
log.log("[止损-监控] {} 进入监控状态 (成本:{:.2f} 现价:{:.2f} 亏损:{:.2f}%)".format(
    code, open_price, current_price, loss_ratio*100))
```

**效果**：
- 符合项目日志规范
- 提高代码一致性
- 便于维护

---

### **3. 优化心跳模块风控检查频率** ✅
**文件**: [`utils/heartbeat.py`](file://d:\mpython\utils\heartbeat.py)

**修改内容**：

| 检查项 | 修改前频率 | 修改后频率 | 说明 |
|-------|----------|----------|------|
| 止损检查 | 每 5 秒 | 每 15 秒 | 避免频繁检查，降低系统开销 |
| 止盈检查 | 每 10 秒 | 每 15 秒 | 与止损保持一致 |
| 时间判断 | "0952" | "0951" | 与动态止盈模块内部时间检查保持一致 |

**效果**：
- 降低系统开销（从每秒 0.3 次检查降低到每秒 0.13 次）
- 风控响应时间仍然在合理范围内（15秒）
- 时间判断逻辑统一

## 💡 使用建议

### **1. 如何查看个股详情**
如需临时查看某只股票的详细持仓信息，可在 [`main.py`](file://d:\mpython\main.py) 的 [print_account_info()](file://d:\mpython\main.py#L100-L146) 函数中取消注释：
```python
# 【可选】如需查看个股详情，可临时取消注释以下代码
for pos in positions_raw:
    stock_code = pos.get('symbol', 'N/A')
    volume = pos.get('volume', 0)
    log.log("   {}: {} 股".format(stock_code, volume))
```

### **2. 如何调整风控检查频率**
在 [`utils/heartbeat.py`](file://d:\mpython\utils\heartbeat.py) 中修改：
```python
# 止损检查间隔（秒）
if current_ts - self.last_stop_loss_check >= 15:  # 可改为 10 或 20

# 止盈检查间隔（秒）
if current_ts - self.last_take_profit_check >= 15:  # 可改为 10 或 20
```

### **3. 风控模块执行时间**
- **止损**: 10:45 - 14:50（由 `settings.STOP_LOSS_START_TIME` 和 `STOP_LOSS_END_TIME` 控制）
- **止盈**: 09:51 之后（由 `DynamicTakeProfit.EARLIEST_EXECUTION_TIME` 控制）
