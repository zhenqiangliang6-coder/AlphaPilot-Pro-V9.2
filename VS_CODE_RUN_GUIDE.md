## 🛠️ 常见问题

### Q1: 报错 `无效的ACCOUNT_ID`？

**原因**: 掘金终端中账户未连接

**解决**:
1. 打开掘金量化终端
2. 进入"账户管理"
3. 确认 "Alphapilot量化智能学习" 状态为"已连接"
4. 如果未连接，点击"连接"
5. 重新启动策略

---

### Q2: 报错 `ModuleNotFoundError: No module named 'gm'`？

**原因**: 虚拟环境未激活或缺少依赖

**解决**:
```powershell
# 1. 激活虚拟环境
.\quant_env\Scripts\Activate.ps1

# 2. 安装依赖
pip install gm python-dotenv watchdog

# 3. 重新启动策略
python main.py --mode live
```

---

### Q3: 策略启动但没有心跳日志？

**可能原因**:
1. `init()` 函数中有错误导致策略未完全初始化
2. 订阅的股票代码有误
3. Token无效

**解决**:
1. 检查控制台是否有错误信息
2. 查看 [logs/](file://d:\mpython\logs) 目录下的日志文件
3. 检查 [config/settings.py](file://d:\mpython\config\settings.py) 配置

---

## 📝 配置说明

### 关键配置文件

| 文件 | 说明 | 关键配置 |
|------|------|---------|
| [.env](file://d:\mpython\.env) | 环境变量 | `GM_TOKEN`, `GM_ACCOUNT_ID` |
| [config/settings.py](file://d:\mpython\config\settings.py) | 策略配置 | `SUBSCRIBE_SYMBOLS`, `RUN_MODE` |
