# 延时策略买入失败问题修复报告

**日期**: 2026-05-13  
**作者**: Alphapilot智能体团队  
**问题类型**: BUG修复 - 延时策略未定时执行  

---

## 📋 问题描述

用户反馈：延时策略观察名单中有股票的目标日是今天(2026-05-13)，但程序没有执行买入操作，尽管账户有足够资金。

日志显示只有两只过期股票被删除：
```
2026-05-13 07:13:41 | INFO | [延时策略-已过期] SZSE.301165 目标日=2026-05-12, 今天=2026-05-13, 自动删除(不再买入)
2026-05-13 07:13:45 | INFO | [延时策略-已过期] SZSE.002046 目标日=2026-05-12, 今天=2026-05-13, 自动删除(不再买入)
```

但没有看到任何买入执行的日志。

---

## 🔍 根本原因分析

### 1. 股票代码格式检查 ✅（通过）

观察名单中的股票代码格式：
- `SZSE.002240` (盛新锂能)
- `SZSE.002679` (福建金森)
- `SZSE.002498` (汉缆股份)
- `SZSE.000537` (绿发电力)
- `SHSE.603538` (美诺华)
- `SZSE.301419` (瑞可达)

这些格式与掘金SDK要求的格式完全一致，**不存在代码格式问题**。

### 2. 延时策略执行机制缺陷 ❌（核心问题）

通过分析代码发现：

**main.py 第320行**：
```python
delayed_strat.check_and_execute()
```

这行代码**只在程序启动时执行一次**，之后没有任何定时任务再次调用它！

对比止损监控的实现：
- `StopLossMonitor` 被注入到 `HeartbeatMonitor`
- 每个心跳周期（每15秒）都会执行止损检查

但 **`DelayedStrategy` 没有被集成到任何定时任务中**！

### 3. 实际影响

假设程序在 07:13 启动：
1. 启动时调用 `check_and_execute()`
2. 此时未到交易时间，量比数据可能不可用
3. 未到14:39保底买入时间
4. **之后再也没有调用过 `check_and_execute()`**
5. 即使到达14:39，也不会自动执行买入

---

## ✅ 解决方案

### 修改1: HeartbeatMonitor 增加延时策略支持

**文件**: `utils/heartbeat.py`

#### 1.1 构造函数添加参数
```python
def __init__(self, log_func, account_info_func, stop_loss_mon=None, take_profit_mon=None, delayed_strat=None):
    """
    参数:
        ...
        delayed_strat: 延时策略实例（可选）
    """
    self.delayed_strat = delayed_strat  # 【新增】延时策略实例
    self.last_delayed_check = 0  # 【新增】延时策略上次检查时间
```

#### 1.2 心跳循环中添加定时检查
```python
# 5. 【新增】延时策略检查（每30秒，确保及时执行买入）
if self.delayed_strat and (current_ts - self.last_delayed_check >= 30):
    try:
        self.delayed_strat.check_and_execute()
        self.last_delayed_check = current_ts
    except Exception as e:
        self.log("[警告] 延时策略检查失败: {}".format(e))
```

### 修改2: main.py 注入延时策略实例

**文件**: `main.py`

```python
# ==================== 第七步：初始化心跳监控器（含风控检查）====================
heartbeat_monitor = HeartbeatMonitor(
    log.log, 
    print_account_info,
    stop_loss_mon=stop_loss_mon,
    take_profit_mon=take_profit_mon,
    delayed_strat=delayed_strat  # 【新增】注入延时策略，实现定时检查
)
heartbeat_monitor.start()
```

## 🎯 技术要点总结

### 架构设计原则
1. **定时任务集中管理**: 所有周期性任务都应该集成到 `HeartbeatMonitor`
2. **避免一次性执行**: 关键业务逻辑必须有重试或定时检查机制
3. **解耦与复用**: 通过依赖注入将策略模块传递给监控器

### 延时策略三阶段控制
```
① target_date 之前 → 禁止买入(无论量比多少)
② target_date 当天 → 两种情况:
   - 情况A: 盘中量比 ≥ trigger_volume_ratio → 立即提前买入 → 删除watchlist
   - 情况B: 盘中量比始终 < trigger_volume_ratio → 14:39保底买入 → 删除watchlist
③ target_date 之后 → 延时策略过期 → 自动删除watchlist(不再买入)
```

### 执行路径
- **路径A（信号优先）**: 量比达到门槛 → 立即买入
- **路径B（保底机制）**: 14:39之后 → 强制买入
