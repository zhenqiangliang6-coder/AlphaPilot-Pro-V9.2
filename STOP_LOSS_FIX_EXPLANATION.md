## 🔴 问题描述

用户反馈：**很多个股下跌到-10%了依然没有触发止损卖出**

---

## 🔍 根本原因分析

### 旧架构的问题

之前的止损检查逻辑在 `on_bar()` 回调函数中执行：

```python
def on_bar(context, bars):
    # 每5秒执行一次止损检查
    if current_ts - last_stop_loss_check >= settings.STOP_LOSS_CHECK_INTERVAL:
        stop_loss_mon.check()
```

**致命缺陷**：
1. `on_bar()` 只在**订阅的股票**有K线数据推送时才会触发
2. 当前配置只订阅了 `SHSE.601138`（工业富联）
3. **其他持仓股票未订阅** → 收不到bar数据 → `on_bar()` 不触发 → 止损不执行

### 为什么下跌-10%也没卖出？

即使你的持仓股票亏损严重，但由于：
- ❌ 这些股票不在 `SUBSCRIBE_SYMBOLS` 列表中
- ❌ 没有K线数据推送
- ❌ `on_bar()` 回调从未被调用
- ❌ 止损检查代码根本没有执行

---

## ✅ 解决方案

### 核心思路

将止损/止盈检查从 `on_bar()` **移至独立的心跳线程**，使其：
- ✅ 不依赖特定股票的行情数据推送
- ✅ 每5秒自动检查所有持仓
- ✅ 确保风控逻辑始终运行

### 修改内容

#### 1. 修改 `utils/heartbeat.py`

**新增功能**：
```python
class HeartbeatMonitor:
    def __init__(self, log_func, account_info_func, 
                 stop_loss_mon=None, take_profit_mon=None):
        """接收止损和止盈监控器实例"""
        self.stop_loss_mon = stop_loss_mon
        self.take_profit_mon = take_profit_mon
        
    def _heartbeat_loop(self):
        while self.running:
            time.sleep(5)
            
            # 【新增】止损检查（每5秒）
            if self.stop_loss_mon:
                self.stop_loss_mon.check()
            
            # 【新增】动态止盈检查（每10秒）
            if self.take_profit_mon and now_time >= "0952":
                self.take_profit_mon.check()
```

#### 2. 修改 `main.py`

**初始化时传入风控实例**：
```python
# 第七步：初始化心跳监控器（含风控检查）
heartbeat_monitor = HeartbeatMonitor(
    log.log, 
    print_account_info,
    stop_loss_mon=stop_loss_mon,      # ✅ 传入止损监控器
    take_profit_mon=take_profit_mon   # ✅ 传入止盈监控器
)
heartbeat_monitor.start()
```

**清理 `on_bar()` 中的冗余代码**：
```python
def on_bar(context, bars):
    """
    K线数据更新回调
    
    职责：
    - 集合竞价策略执行
    - 延时策略和火箭加仓的时间触发检查
    - 【注意】止损/止盈已移至心跳线程独立执行，不再依赖bar数据
    """
    # ... 移除了止损/止盈检查逻辑
```

## 📊 修复效果对比

| 项目 | 修复前 | 修复后 |
|------|--------|--------|
| **执行位置** | `on_bar()` 回调 | 心跳线程 |
| **依赖条件** | 需要订阅股票行情 | 无需订阅，检查所有持仓 |
| **执行频率** | 仅当有bar数据时 | 固定每5秒 |
| **覆盖范围** | 仅订阅的股票 | 所有持仓股票 |
| **可靠性** | ❌ 低（依赖外部数据） | ✅ 高（独立运行） |

```python

STOP_LOSS_END_TIME = "1450"
```

## ⚠️ 重要提示

### 1. 止损阈值配置

当前配置（保守版 V8.95）：
- **监控触发**：亏损 ≥ **-0.8%** 开始跟踪
- **一级止损**：亏损 ≥ **-1.7%** 卖出50%仓位
- **二级止损**：亏损 ≥ **-3.5%** 全部清仓

如果你的股票下跌-10%，应该已经触发二级止损。如果仍未卖出，请检查：
- 持仓成本价是否正确（必须 > 0）
- 是否有可用持仓数量（`can_use_volume > 0`）
- 当前时间是否在 10:45-14:50 之间

### 2. 日志检查

查看 `logs/` 目录下的最新日志文件，搜索关键词：
- `[止损-监控]` - 进入监控状态
- `[止损-一级]` - 触发一级止损
- `[止损-二级]` - 触发二级止损
- `[止损失败]` - 止损执行失败的原因

### 3. 调试建议

如果修复后仍无止损日志输出：
1. 确认程序已重启（修改代码后必须重启）
2. 确认当前时间在止损时间窗口内
3. 确认持仓股票的成本价 > 0
4. 手动触发一次检查（可在程序中临时添加测试代码）

## 📝 技术细节

### 为什么选择心跳线程？

1. **独立性**：不依赖外部数据源（行情推送）
2. **稳定性**：固定频率执行，不受市场数据影响
3. **全面性**：可以检查所有持仓，无论是否订阅
4. **解耦性**：风控逻辑与交易逻辑分离

### 性能影响

- 每5秒检查一次持仓（默认配置）
- 每次检查遍历所有持仓股票
- 对系统性能影响极小（通常 < 10ms）
